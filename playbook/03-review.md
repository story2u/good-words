# playbook/03 · 校对闸门与入库

校对者（Reviewer）逐卡核验候选摘抄卡，过闸门后定稿。

## 选角

Reviewer 必须与该书 Reader **不同模型**（HARNESS.md 红线）。

## Showrunner 组 prompt（自包含）

包含：

1. 角色设定：你是校对员，逐卡核验，不重写、不增删摘抄内容；
2. 待审卡片全文 + 对应范围的原文文本（供逐字比对）；
3. 闸门清单（HARNESS.md §6 G1–G4 全文）+ 该分类标签表；
4. 输出格式：每卡一行结论 `通过 / 退回(原因+闸门编号)`，末尾给本批次总评。

## 落盘

1. 通过的卡：`状态: 候选 → 定稿`；
2. 退回的卡：标 `状态: 退回(Gx)`，留给下一轮 02 修正或删除；
3. 在 `excerpts/<书名>.md` 末尾「审校记录」追加一行：日期、批次、Reviewer 模型、通过/退回数、要点；
4. 全书批次完成且无退回卡 → `books.md` 状态置 `已完成`；
5. 更新 `relay/HANDOFF.md`。

## G4 触发

累计卡数逼近指导线（40 卡或全书 2%）时，Showrunner 暂停该书并请作者裁决：收紧选卡标准或特批扩容。

# HARNESS.md · 宪法

本文件是 good-words 阅读摘抄 harness 的最高规则。与其他文档冲突时，以本文件为准。

---

## 1. 使命与产出

让 AI Agent 读好书，**分类摘抄、提炼好词好句**。最终产出是按分类组织的摘抄卡片库：

- 每条摘抄＝一张「摘抄卡」：原文 + 出处 + 提炼 + 标签；
- 每本书＝一份「书目档案」+ 一个摘抄文件；
- 每个分类＝`categories/` 下一个目录，含分类说明与书单。

## 2. 角色

| 角色 | 担任者 | 职责 |
|---|---|---|
| **Showrunner** | Claude 主会话 | 组 prompt、调度、守闸门、落盘记账；不代演 |
| **阅读者 Reader** | claude / ChatGPT / Gemini 任一 | 通读指定范围，产出候选摘抄卡 |
| **校对者 Reviewer** | 与 Reader **不同模型** | 按闸门逐卡核验，给出 通过/退回 |

运行方式默认 **方式 M·手动 CLI 接力**（沿用 ai-story 经验）：Showrunner 组好自包含 prompt，作者在终端手动执行 `claude -p` / `codex exec` / `gemini` 等命令并把结果带回，Showrunner 校验落盘。条件允许时可升级为 Showrunner 直接 Bash 调用。

## 3. 文档即状态

- `relay/HANDOFF.md` 是唯一接力板：当前书目、批次、待办、下一步。任何会话开始先读它，结束必须更新它。
- 书的生命周期状态只记在 `categories/<分类>/books.md` 的状态列：`候选 → 入库 → 摘抄中 → 校对中 → 已完成`。
- 所有产物落盘为 Markdown，不依赖任何会话记忆。

## 4. 摘抄规则

1. **忠实**：逐字照录所选译本，不改写、不润色、不混搭不同译本；省略中间内容用「……」并保证不歪曲原意。
2. **可溯源**：每卡必须标 *书名、版本（译者/出版社）、章节*；赛斯口述书**必须标节号（session）**，有页码更好。
3. **单卡一意**：一张卡只承载一个完整意思，长度以一句至一小段为宜（一般 ≤150 字，确需长段须在提炼中说明理由）。
4. **提炼**：每卡附 1–2 句提炼——它好在哪、说了什么、适合什么场景引用。提炼是摘抄者自己的话，不得伪装成原文。
5. **语言**：以中文译本为准摘抄；书单中标注英文原名以便核对，必要时可在卡内附原文对照。

## 5. 分类与标签

- **一级分类**＝`categories/` 目录（现有：新时代）。新建分类需建 `README.md`（定义+收录标准）与 `books.md`（书单）。
- **主题标签**＝卡内 `tags`。每个分类在其 README 维护标签表；新增标签要在标签表登记一行说明，避免同义滥生。

## 6. 质量闸门（校对者逐卡核验）

| 闸门 | 内容 | 不过则 |
|---|---|---|
| **G1 出处可溯** | 版本/章节/节号齐全且真实存在 | 退回补全 |
| **G2 抄录忠实** | 与所标版本原文逐字一致，省略号使用规范 | 退回修正 |
| **G3 归类合理** | 分类、标签与卡片内容相符，标签在标签表内 | 退回改标 |
| **G4 摘录克制** | 单本累计摘录克制（指导线：≤40 卡或全书正文 2%，以先到为准；超出需作者特批） | 暂停该书摘抄 |

校对产物记入该书摘抄文件末尾的「审校记录」，注明校对模型与结论。

## 7. 版权与使用边界

摘抄仅供**个人学习研究**，不公开传播成规模的原文集合；引用时保留出处。G4 的摘录克制条款即为此设。

## 8. 已知坑（继承自 ai-story）

- `claude -p` 子进程会加载本仓库 CLAUDE.md，可能发生身份混淆 → CLAUDE.md 已设「主会话/子进程分流」条款；再混淆则改用自包含 prompt + 空目录执行。
- 演员 prompt 不含仓库路径、不依赖会话状态，**全量自包含重发**。
- 评审与 maker 不同模型，这是闸门有效性的底线。
